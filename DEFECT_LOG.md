## Notes on Patterns

**CASE_CAPTION.json schema drift (D-03, D-04):** Every new job folder gets a hand-built CASE_CAPTION.json. If the schema evolves and there's no migration, new jobs break silently. Normalizers in config.py buy time — but a canonical CASE_CAPTION template and onboarding checklist would prevent recurrence.

**One-format assumptions (D-01, D-02):** Both exhibit parsers assumed one steno format. The same assumption likely exists in other pattern-matching code. Before onboarding a third CR, audit all regex patterns that parse steno structure for similar single-format assumptions.

**Regression reference data gap (D-05, D-06):** Easley CASE_CAPTION.json was hand-built before current schema stabilized. Regression passes now but Easley data is a known degraded reference. Consider rebuilding Easley CASE_CAPTION to full schema before next regression expansion.
